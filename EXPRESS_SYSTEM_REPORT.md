# 快递管理系统汇报

---

## 📋 项目概述

### 项目名称
快递管理系统 (Express Management System)

### 项目目标
- 实现快递信息的增删查改功能
- 追踪快递的领取状态
- 提供数据统计和分析
- 支持数据持久化存储

### 技术栈
- **语言**: C语言
- **平台**: Windows/Linux跨平台
- **存储**: 二进制文件存储
- **编码**: UTF-8编码支持

---

## 🎯 核心功能

### 1. 添加快递 (Add Express)
- ✅ 自动生成快递编号 (EXP000001格式)
- ✅ 输入收件人信息（姓名、电话）
- ✅ 记录寄件人信息
- ✅ 自动记录收货时间戳
- ✅ 输入验证和错误处理

### 2. 查询快递 (Query Express)
- ✅ 按快递编号查询
- ✅ 按收件人姓名查询
- ✅ 列出所有快递记录
- ✅ 详细信息展示

### 3. 标记为已领取 (Mark as Picked Up)
- ✅ 标记快递为已领取
- ✅ 自动记录领取时间
- ✅ 防止重复标记
- ✅ 确认消息提示

### 4. 删除快递 (Delete Express)
- ✅ 仅删除已领取的快递
- ✅ 防误删机制
- ✅ 数组动态调整

### 5. 统计信息 (Statistics)
- ✅ 快递总数统计
- ✅ 已领取/未领取统计
- ✅ 领取率百分比计算
- ✅ 实时数据展示

### 6. 数据管理 (Data Management)
- ✅ 保存数据到文件
- ✅ 启动时自动加载数据
- ✅ 序号防重复机制
- ✅ 异常恢复机制

## 🏗️ 系统架构

### 数据结构

```c
typedef struct {
    char id[20];              // 快递编号
    char recipient_name[50];  // 收件人
    char phone[20];          // 电话
    char sender[50];         // 寄件人
    char address[100];       // 地址
    char receive_time[30];   // 收货时间
    int is_received;         // 领取标志
    char pickup_time[30];    // 领取时间
} Express;
```

### 全局变量

| 变量 | 类型 | 说明 |
|------|------|------|
| `express_db[1000]` | Express数组 | 快递数据库 |
| `express_count` | int | 当前快递数量 |
| `next_serial` | int | 下一个序号 |

### 主要函数

| 函数 | 功能 |
|------|------|
| `add_express()` | 添加新快递 |
| `query_by_id()` | 按编号查询 |
| `query_by_name()` | 按姓名查询 |
| `mark_as_received()` | 标记已领取 |
| `delete_express()` | 删除快递 |
| `statistics()` | 统计分析 |
| `save_to_file()` | 保存数据 |
| `load_from_file()` | 加载数据 |

## 💾 数据存储

### 存储格式
- **文件名**: `express_data.dat`
- **格式**: 二进制格式
- **内容**: 快递数量 + 快递数据

### 加载流程
1. 读取快递数量
2. 验证数量合法性
3. 读取快递数据
4. 初始化序号生成器
5. 防止ID重复
